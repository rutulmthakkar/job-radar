## Fix 1: Strip UTF-8 BOMs

Multiple files were written by PowerShell `Set-Content -Encoding UTF8`,
which prepends a BOM. This is breaking CSV parsing (`KeyError: 'company'`).
Strip BOMs from every `.py`, `.csv`, `.md`, `.json`, `.html`, and `.yml`
file in the repo. Verify by checking the first bytes of `build.py` and
`data/perm_sponsors.csv` — they should not start with `0xEF 0xBB 0xBF`.

## Fix 2: Filter jobs to US + Canada only

In `fetchers/jobs.py`, after each ATS fetcher (`_greenhouse`, `_lever`,
`_ashby`) returns its job list, filter out any role whose `location`
field doesn't look US- or Canada-based.

Build a small helper `_is_us_or_canada(location: str) -> bool`:

- Return True if the string contains a US state code (CA, NY, WA, TX, MA,
  IL, CO, GA, FL, OR, VA, NC, NJ, PA, OH, MI, AZ, MN, WI, MO, IN, TN, MD,
  CT, NV, UT, KS, AR, MS, NM, NE, ID, HI, ME, MT, RI, DE, SD, ND, AK, VT,
  WY, WV, DC) or US city names (San Francisco, New York, Seattle, Austin,
  Boston, Chicago, Denver, Atlanta, Los Angeles, Portland, etc.), or any
  variant of "United States", "USA", "US".
- Return True if it contains "Canada" or a Canadian province code (ON,
  BC, QC, AB, MB, SK, NS, NB, NL, PE).
- Return True for "Remote" alone, but False for "Remote, India" or
  "Remote - EMEA" or anything that names a non-US/non-CA country.
- Return False if the string explicitly names a non-US/CA country
  (India, UK, Britain, Germany, France, Spain, Brazil, Mexico, Japan,
  China, Singapore, Australia, Israel, Ireland, Netherlands, Poland,
  Argentina, Colombia, etc.).
- When empty/ambiguous, return True (be permissive).

Apply the filter in `fetch()` before returning.
